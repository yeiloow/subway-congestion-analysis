## 필수 구현 요소

1. 데이터 수집: 최소 2개 이상 소스 (API + 크롤링)
2. 데이터베이스: SQLite or PostgreSQL, 정규화된 스키마 설계
3. 텍스트 분석: 감정분석 OR 토픽모델링 OR 키워드 추출
4. 시각화: Plotly 인터랙티브 대시보드 5개 이상
5. 최종 리포트: Jupyter Notebook + README.md

## 팁

### 데이터 수집 팁

- ✅ API 우선 고려: 크롤링보다 안정적이고 빠름
- ✅ robots.txt 확인: 크롤링 전 반드시 확인
- ✅ 에러 처리: try-except로 안전한 수집
- ✅ Rate Limiting: API 호출 제한 준수 (time.sleep())
- ✅ 백업: 수집 데이터는 주기적으로 저장

### 팀워크 팁

- ✅ 역할 분담: 데이터 수집/분석/시각화/발표
- ✅ Git 사용: Branch 전략, Commit 메시지 규칙
- ✅ 일일 Standup: 진행 상황 공유
- ✅ 문서화: README, 주석, 기술 블로그
- ✅ 코드 리뷰: 서로 코드 리뷰 및 피드백

### 발표 팁

- ✅ 스토리텔링: 문제 → 분석 → 인사이트 → 가치
- ✅ 시각화 중심: 텍스트보다 그래프
- ✅ 라이브 데모: 실제 작동하는 모습 시연
- ✅ 리허설: 발표 연습 최소 2회
- ✅ Q&A 준비: 예상 질문 리스트
